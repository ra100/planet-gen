# Tectonic Plate Simulation for Procedural Planet Generation

**Research date:** 2026-03-27

---

## 1. Procedural Plate Tectonics -- Overview of Approaches

Three dominant paradigms exist for procedural plate tectonics in game/graphics applications:

### A. Physics-Lite / Rule-Based Expansion
Random seed points are placed on a sphere or 2D plane. Plates grow outward via **flood-fill BFS** with cost functions controlling shape. Boundary interactions are classified by comparing plate velocity vectors and mapped to terrain features via lookup tables. This is fast (seconds) but physically shallow.

**Representative implementations:** Red Blob Games (2018), ProcGenesis, SqueakySpacebar (2017).

### B. Convection-Driven Simulation
Mantle convection cells are modeled (often as a heat map), and plates move as a consequence of convective forces. Plate boundaries emerge from the simulation rather than being imposed. More physically grounded but more expensive.

**Representative implementations:** Cauliflower Labs (2014), SimpleTectonics / weigert (2020), tectonics.js.

### C. Procedural Approximation of Geological Time
Plate movement is simulated over discrete time steps. At each step: move plates, detect collisions, deform lithosphere (uplift/subduction), erode. The simulation runs for hundreds of steps to produce a final heightmap.

**Representative implementations:** Cortial et al. (2019), PlaTec/Mindwerks, Cordonnier et al. (2016).

## 2. Voronoi-Based Plate Generation

Nearly every implementation reviewed uses Voronoi diagrams for spatial subdivision:

### Approach
1. **Generate seed points** on the surface (random, Fibonacci sphere, or Poisson disc sampling).
2. **Compute Voronoi tessellation** -- each cell is a region. On a sphere, the 3D convex hull of the points yields the spherical Delaunay triangulation; its dual is the spherical Voronoi diagram.
3. **Assign cells to plates** by selecting N plate origins (typically 5-50) and expanding outward.

### Key Techniques

| Technique | Purpose | Source |
|-----------|---------|--------|
| **Lloyd's relaxation** | Makes cells more uniform; prevents degenerate thin slivers | Red Blob Games |
| **Random-fill flood** (randomized queue) | Produces jagged, natural-looking plate boundaries vs. strict BFS | Red Blob Games |
| **Farthest-point seeding** with compactness constraints | Avoids plates that wrap around the sphere | World Orogen |
| **H3 hex tiles** (~41k regions) | Uniform area cells with reliable neighbor queries | kenny.wtf |
| **Fibonacci sphere + Delaunay** | Efficient near-uniform point distribution on sphere | World Orogen, Cortial et al. |

### Pitfall
Pure Voronoi plate boundaries look too regular and sharp. Solutions include (a) random-fill instead of BFS, (b) noise perturbation of boundary edges, and (c) using Voronoi only for small regions, then grouping them into plates.

**Source:** SqueakySpacebar (2017) -- "plates generated by simple Voronoi diagrams were found to be too unnatural-looking, with boundaries too sharply delineated and plate sizes too regular."

## 3. Plate Boundary Types and Their Effect on Terrain

### Classification Algorithm
At each boundary edge between plate A and plate B:
1. Compute the **relative velocity vector** = v_A - v_B
2. Project onto the **boundary normal** n: component_normal = dot(v_rel, n)
3. Project onto the **boundary tangent** t: component_tangent = dot(v_rel, t)

- **Convergent:** component_normal < -threshold (plates approaching)
- **Divergent:** component_normal > +threshold (plates separating)
- **Transform:** |component_tangent| >> |component_normal|

Red Blob Games uses a simpler threshold: convergent if delta_distance < -0.75 (empirical).

### Boundary Effects on Elevation

| Boundary Type | Crust Combination | Terrain Feature | Typical Elevation Range |
|---------------|-------------------|-----------------|------------------------|
| **Convergent** | Continental + Continental | Fold mountains (Himalayas) | +4,000 to +8,849 m |
| **Convergent** | Oceanic + Continental | Subduction zone, coastal mountains, volcanic arc | Mountains: +2,000-6,000 m; Trench: -8,000 to -11,000 m |
| **Convergent** | Oceanic + Oceanic | Island arc + deep trench | Islands: +500-2,000 m; Trench: -6,000 to -11,000 m |
| **Divergent** | Oceanic + Oceanic | Mid-ocean ridge | Ridge crest: -2,000 to -2,500 m (above -4,000 m abyssal floor) |
| **Divergent** | Continental + Continental | Rift valley (East African Rift) | Valley floor: +400-600 m; Flanks: +2,000-3,000 m |
| **Transform** | Any | Strike-slip fault, minimal elevation change | Negligible vertical offset; lateral displacement |
| **Hotspot** | Through any plate | Shield volcano / volcanic island chain | +0 to +4,200 m (Mauna Kea from sea floor: ~10,200 m) |

### Earth's Bimodal Hypsographic Distribution
The bimodal elevation distribution is a key target for realistic generation:
- **Continental peak:** ~+100 m (lowland plains)
- **Oceanic peak:** ~-4,700 m (abyssal plains)
- **Continental crust:** average thickness 35 km, density 2.7-2.8 g/cm^3 (granitic)
- **Oceanic crust:** average thickness 5 km, density 3.0 g/cm^3 (basaltic)
- **Land average:** +840 m; covers 29% of surface
- **Ocean average:** -3,800 m; covers 71% of surface

**Source:** Hypsometry (Wikipedia); NOAA ETOPO1 hypsographic curve.

## 4. Continental Drift Simulation

### Time-Stepping Model (PlaTec / Cauliflower Labs)
Each time step:
1. **Move plates** according to velocity vectors (or convection-derived forces)
2. **Detect overlaps** (convergent) and **gaps** (divergent)
3. At overlaps: subduct denser plate under lighter; add mass/height to overriding plate
4. At gaps: spawn new oceanic crust
5. Apply **erosion** (optional: stream-power model)
6. Check for **supercontinent cycle** triggers (optional)

**Cauliflower Labs** sums convection cell vectors beneath each plate to compute net plate velocity. PlaTec uses ~0.29 Myr average time steps on a 500k-cell world (~0.7 s/step compute time).

### Supercontinent Cycle
Some simulations model the Wilson Cycle: plates converge into a supercontinent, which then rifts apart due to trapped mantle heat. Cauliflower Labs models this explicitly via hotspot weakening of supercontinents.

## 5. Plate Velocities and Euler Poles

### Euler's Fixed Point Theorem
Any rigid-body motion on a sphere can be described as rotation about an axis (the Euler pole). A plate's motion is fully specified by:
- **Euler pole location** (latitude_p, longitude_p)
- **Angular velocity** omega (degrees/Myr)

### Velocity Equation
The linear velocity at any point on the plate:

```
v = omega * R * sin(theta)
```

Where:
- v = linear velocity (cm/yr if omega in appropriate units)
- R = Earth's radius (6,371 km)
- theta = angular distance from the point to the Euler pole

Velocity is **zero** at the Euler pole and **maximum** at 90 degrees from it.

### Full Computation (from UC Davis Geophysics)
Given a point at (lambda_x, phi_x) and Euler pole at (lambda_p, phi_p):

1. Angular distance: `a = arccos[sin(lx)*sin(lp) + cos(lx)*cos(lp)*cos(phi_p - phi_x)]`
2. Azimuth: `C = arcsin[cos(lp)*sin(phi_p - phi_x) / sin(a)]`
3. Velocity magnitude: `v = omega * R * sin(a)`
4. Motion azimuth: `beta = 90 + C`

### Real-World Example
Nazca-South America boundary at 28S, 71W:
- Euler pole: 56N, 94E
- Angular velocity: 7.6 x 10^-7 deg/yr
- Computed velocity: **8.43 cm/yr**
- Azimuth: 77.35 degrees from north

### Combining Plates
To find A-C motion from A-B and B-C Euler vectors: convert to Cartesian, add vectorially, reconvert to spherical.

**Source:** UC Davis GEL 056 Geophysics course (LibreTexts).

## 7. Elevation from Plate Boundaries -- Algorithmic Details

### World Orogen Approach (most detailed open-source implementation found)
Elevation is computed by combining multiple distance fields:

1. **Mountain distance field:** Distance from convergent boundary edges, with stress-driven uplift scaling.
2. **Ocean distance field:** Distance from plate interior toward coast.
3. **Coastline distance field:** Distance from coast in both directions.

These are blended via a **harmonic-mean formula** to produce smooth transitions. Additional layers:
- **Asymmetric mountain profiles** (steep subduction side, gentle back-arc side)
- **Continental shelf** profiles (gradual slope to -150 m, then steeper continental slope)
- **Graben/rift profiles** at divergent continental boundaries
- **Back-arc basin** depressions behind subduction zones
- **Island arc** elevation bumps at oceanic-oceanic convergence
- **Hotspot volcanism** with drift-trail island chains
- **Multi-layered fBm noise** for detail at all scales

### SqueakySpacebar Approach (stress-based)
Each plate gets a random force vector. For boundary edges:
- **Compressive stress** = dot product of combined plate forces with edge normal
- **Shear stress** = scalar product with edge tangent
- Uplift propagates from boundary via flood-fill with exponential decay
- Elevation mapped to [-11,000 m, +9,000 m] with sea level at normalized 0.55

### SimpleTectonics Buoyancy Model (physics-based)
Segment floating height derived from isostasy:
```
d_above = d * (1 - rho)        // height above surface
d_below = m / A                 // depth below surface
rho = m / (A * d)               // density
```
Where m = mass, A = area, d = thickness. Height cascading between neighbors uses friction parameter maxdiff = 0.01 and transport rate = 0.2.

### Crystallization Growth Rate (SimpleTectonics)
New crust at divergent boundaries grows according to:
```
G = k_G * (1 - T) * (1 - T - d_below)
```
Equilibrium deposit density (Langmuir isotherm):
```
D = k_D * (1 - T) / (1 - k_D * (1 - T))
```
Where T = temperature, k_G and k_D are rate constants.

- **License:** LGPL-3.0

## 9. Summary of Key Algorithms and Equations

### Euler Pole Velocity
```
v = omega * R * sin(theta)
```

### Stream Power Erosion (Cordonnier et al. 2016)
```
dh/dt = U(x) - K * A^m * |grad(h)|^n
```
Typical: m=0.5, n=1, K depends on rock type and climate.

### Isostatic Buoyancy (SimpleTectonics)
```
floating_height = thickness * (1 - density)
submerged_depth = mass / area
```

### Crystallization at Ridges (SimpleTectonics)
```
G = k_G * (1-T) * (1-T-d_below)
D_eq = k_D*(1-T) / (1 - k_D*(1-T))
```

### Boundary Classification (general)
```
v_rel = v_A - v_B
normal_component = dot(v_rel, boundary_normal)
tangent_component = dot(v_rel, boundary_tangent)

if normal_component < -threshold: CONVERGENT
elif normal_component > +threshold: DIVERGENT
else: TRANSFORM
```

### Elevation Scaling (SqueakySpacebar)
```
elevation_meters = (normalized_value - 0.55) * 20000
// maps [0,1] to [-11,000m, +9,000m] with sea level at 0.55
```
