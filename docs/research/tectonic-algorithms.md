## 1. Plate Generation Algorithms

### 1.1 Voronoi-Based Spatial Subdivision

Nearly every reviewed implementation uses Voronoi diagrams for spatial subdivision:

1. **Generate seed points** on the surface (random, Fibonacci sphere, or Poisson disc sampling).
2. **Compute Voronoi tessellation** -- on a sphere, the 3D convex hull of the points yields the spherical Delaunay triangulation; its dual is the spherical Voronoi diagram.
3. **Assign cells to plates** by selecting N plate origins (typically 5-50) and expanding outward via flood-fill.

### 1.2 Key Techniques

| Technique                                               | Purpose                                                          | Source                       |
| ------------------------------------------------------- | ---------------------------------------------------------------- | ---------------------------- |
| **Lloyd's relaxation**                                  | Makes cells more uniform; prevents degenerate thin slivers       | Red Blob Games               |
| **Random-fill flood** (randomized queue)                | Produces jagged, natural-looking plate boundaries vs. strict BFS | Red Blob Games               |
| **Farthest-point seeding** with compactness constraints | Avoids plates that wrap around the sphere                        | World Orogen                 |
| **H3 hex tiles** (~41k regions)                         | Uniform area cells with reliable neighbor queries                | kenny.wtf                    |
| **Fibonacci sphere + Delaunay**                         | Efficient near-uniform point distribution on sphere              | World Orogen, Cortial et al. |

### 1.3 Pitfall: Overly Regular Boundaries

Pure Voronoi plate boundaries look too regular and sharp. Solutions:

- Random-fill instead of BFS (randomized priority queue)
- Noise perturbation of boundary edges
- Use Voronoi only for small regions, then group them into plates

**Source:** SqueakySpacebar (2017) -- "plates generated by simple Voronoi diagrams were found to be too unnatural-looking, with boundaries too sharply delineated and plate sizes too regular."

## 2. Boundary Classification Algorithms

### 2.1 Velocity-Based Classification

At each boundary edge between plate A and plate B:

1. Compute the **relative velocity vector**: `v_rel = v_A - v_B`
2. Project onto the **boundary normal** n: `component_normal = dot(v_rel, n)`
3. Project onto the **boundary tangent** t: `component_tangent = dot(v_rel, t)`

Classification:

```
if component_normal < -threshold:  CONVERGENT  (plates approaching)
elif component_normal > +threshold: DIVERGENT   (plates separating)
else:                                TRANSFORM   (|tangent| >> |normal|)
```

Red Blob Games uses a simpler empirical threshold: convergent if `delta_distance < -0.75`.

### 2.2 Boundary Effects on Elevation

| Boundary Type  | Crust Combination         | Terrain Feature                                  | Typical Elevation Range                                |
| -------------- | ------------------------- | ------------------------------------------------ | ------------------------------------------------------ |
| **Convergent** | Continental + Continental | Fold mountains (Himalayas)                       | +4,000 to +8,849 m                                     |
| **Convergent** | Oceanic + Continental     | Subduction zone, coastal mountains, volcanic arc | Mountains: +2,000-6,000 m; Trench: -8,000 to -11,000 m |
| **Convergent** | Oceanic + Oceanic         | Island arc + deep trench                         | Islands: +500-2,000 m; Trench: -6,000 to -11,000 m     |
| **Divergent**  | Oceanic + Oceanic         | Mid-ocean ridge                                  | Ridge crest: -2,000 to -2,500 m                        |
| **Divergent**  | Continental + Continental | Rift valley (East African Rift)                  | Valley floor: +400-600 m; Flanks: +2,000-3,000 m       |
| **Transform**  | Any                       | Strike-slip fault                                | Negligible vertical offset; lateral displacement       |
| **Hotspot**    | Through any plate         | Shield volcano / volcanic island chain           | +0 to +4,200 m (Mauna Kea from sea floor: ~10,200 m)   |

## 3. Euler Poles and Plate Velocity Computation

### 3.1 Euler's Fixed Point Theorem

Any rigid-body motion on a sphere is a rotation about an axis (the Euler pole). A plate's motion is fully specified by:

- **Euler pole location** (latitude_p, longitude_p)
- **Angular velocity** omega (degrees/Myr)

### 3.2 Velocity Equation

```
v = omega * R * sin(theta)
```

Where R = planet radius, theta = angular distance from the point to the Euler pole. Velocity is **zero** at the Euler pole and **maximum** at 90 degrees from it.

### 3.3 Full Computation

Given a point at (lambda_x, phi_x) and Euler pole at (lambda_p, phi_p):

1. Angular distance: `a = arccos[sin(lx)*sin(lp) + cos(lx)*cos(lp)*cos(phi_p - phi_x)]`
2. Azimuth: `C = arcsin[cos(lp)*sin(phi_p - phi_x) / sin(a)]`
3. Velocity magnitude: `v = omega * R * sin(a)`
4. Motion azimuth: `beta = 90 + C`

### 3.4 Worked Example

Nazca-South America boundary at 28S, 71W:

- Euler pole: 56N, 94E
- Angular velocity: 7.6 x 10^-7 deg/yr
- Computed velocity: **8.43 cm/yr**
- Azimuth: 77.35 degrees from north

### 3.5 Combining Plates

To find A-C motion from A-B and B-C Euler vectors: convert to Cartesian, add vectorially, reconvert to spherical.

## 4. Elevation Models from Plate Boundaries

### 4.1 World Orogen Approach (Distance-Field Based)

Elevation is computed by combining multiple distance fields:

1. **Mountain distance field:** Distance from convergent boundary edges, with stress-driven uplift scaling.
2. **Ocean distance field:** Distance from plate interior toward coast.
3. **Coastline distance field:** Distance from coast in both directions.

These are blended via a **harmonic-mean formula** for smooth transitions. Additional layers:

- Asymmetric mountain profiles (steep subduction side, gentle back-arc side)
- Continental shelf profiles (gradual slope to -150 m, then steeper continental slope)
- Graben/rift profiles at divergent continental boundaries
- Back-arc basin depressions behind subduction zones
- Island arc elevation bumps at oceanic-oceanic convergence
- Hotspot volcanism with drift-trail island chains
- Multi-layered fBm noise for detail at all scales

### 4.2 SqueakySpacebar Approach (Stress-Based)

Each plate gets a random force vector. For boundary edges:

- **Compressive stress** = dot product of combined plate forces with edge normal
- **Shear stress** = scalar product with edge tangent
- Uplift propagates from boundary via flood-fill with exponential decay
- Elevation mapped to [-11,000 m, +9,000 m] with sea level at normalized 0.55:
  ```
  elevation_meters = (normalized_value - 0.55) * 20000
  ```

### 4.3 SimpleTectonics Buoyancy Model (Physics-Based)

Segment floating height derived from isostasy:

```
d_above = d * (1 - rho)        // height above surface
d_below = m / A                 // depth below surface
rho = m / (A * d)               // density
```

Where m = mass, A = area, d = thickness. Height cascading between neighbors uses `maxdiff = 0.01` and `transport rate = 0.2`.

### 4.4 Crystallization Growth at Ridges (SimpleTectonics)

New crust at divergent boundaries grows according to:

```
G = k_G * (1 - T) * (1 - T - d_below)
```

Equilibrium deposit density (Langmuir isotherm):

```
D = k_D * (1 - T) / (1 - k_D * (1 - T))
```

Where T = temperature, k_G and k_D are rate constants.

### 4.5 PlaTec Iterative Collision Model

1. Start with flat fractal heightmap, split into plates
2. Move plates each timestep (~0.29 Myr average)
3. Continental collision: fold terrain upward (additive height at collision zones)
4. Subduction: oceanic plate slides under, slight elevation increase
5. Erosion smoothing between steps

Height is built up from the boundary inward over many iterations, creating natural falloff. **Limitation:** Requires hundreds of steps; not suitable for single-pass GPU generation.

## 5. Isostasy and Hypsometry

### 5.1 Airy Isostasy Model

The Airy-Heiskanen model derives elevation from crustal thickness:

```
elevation = thickness * (1 - rho_crust / rho_mantle)
```

For continental crust (rho_c=2.7, rho_m=3.3):

```
elevation = thickness * (1 - 2.7/3.3) = thickness * 0.182
```

A 35 km thick continent stands ~6.4 km above a 7 km thick ocean floor.

**Simplified form for GPU:**

```
h_surface = (thickness - T_ref) * (rho_m - rho_c) / rho_m
```

Where `T_ref` is a reference thickness (the thickness producing sea-level elevation). This is a simple multiply-add, GPU-friendly.

### 5.2 The Puzzle-Piece Problem

**Problem:** When `height ~ f(boundary_dist)`, all points at the same distance from a plate boundary have similar heights. Entire contour rings pop above/below water together as uniform puzzle pieces.

**Solution:** Replace `plate_type -> dome height` with:

```
plate_type + noise -> crustal_thickness -> isostatic_height -> + boundary_features -> final_height
```

Instead of `boundary_dist` controlling height monotonically, **noise controls thickness variation independently of distance from boundary**. Points at the same boundary distance can have wildly different heights.

**Before (dome):**

```
      ___plateau___
     /              \
    /                \  <- uniform height at same boundary_dist
___/                  \___
   ^boundary           ^boundary
```

**After (thickness + noise):**

```
    peak    valley   ridge
     /\      v       /\
    /  \    / \     /  \    <- varied heights everywhere
___/    \__/   \___/    \___
   ^boundary               ^boundary
```

### 5.3 Intra-Plate Thickness Variation

Continental plates have MORE internal variation (basins, plateaus, lowlands); oceanic plates have LESS (mostly uniform abyssal plain).

A continental plate with base 38 km and +/-12 km variation ranges from ~26-50 km thickness. Through isostasy, that is a height range of roughly -2 km to +2.2 km -- some parts underwater, some above. Water level changes reveal terrain gradually.

### 5.4 Hypsometric Curve Fitting

For a planet with two crustal types, the distribution can be modeled as a bimodal Gaussian:

```
f(z) = (f_c / sigma_c) * phi((z - mu_c) / sigma_c)
     + (f_o / sigma_o) * phi((z - mu_o) / sigma_o)
```

Where:

- f_c, f_o = fractional areas of continent/ocean (~0.29, ~0.71 for Earth)
- mu_c, mu_o = mean elevations of each mode
- sigma_c, sigma_o = standard deviations of each mode
- phi = standard normal distribution function

The hypsometric integral (HI) = area under the normalized curve, ranges from 0 (fully eroded) to 1 (uneroded). Earth's global HI ~ 0.45.

### 5.5 Continental Margin Transition Profiles

The transition from continental to oceanic crust should include:

- **0-200 m depth:** Continental shelf (wide, gentle)
- **200-4000 m:** Continental slope (steep)
- **4000 m+:** Abyssal plain

Non-linear transfer function on the margin blend:

```wgsl
let shelf_t = smoothstep(0.02, 0.06, boundary_dist);
let slope_t = smoothstep(0.06, 0.12, boundary_dist);
```

### 5.6 Craton Stability

The oldest parts of continents (cratons) have stable, moderate elevation. A "stability" noise field reduces extreme elevation in plate interiors:

```wgsl
let stability = smoothstep(0.1, 0.3, boundary_dist);
let dampened_variation = intra_plate_variation * mix(1.0, 0.6, stability);
```

### 5.7 Oceanic Crust Age-Depth Relationship

Real ocean floor deepens away from mid-ocean ridges as crust cools and contracts:

```wgsl
let age_depth = boundary_dist * 0.5;
let oceanic_depth_correction = age_depth * (1.0 - f32(is_continental));
height -= oceanic_depth_correction;
```

## 6. WGSL Implementation: Thickness-Based Terrain

### 6.1 Per-Plate Crustal Thickness Base

```wgsl
// Continental: 30-45 km base (above-water terrain)
// Oceanic: 6-9 km base (deep ocean floor)
let base_thickness = select(7.0, 38.0, is_continental);
```

### 6.2 Intra-Plate Thickness Variation

```wgsl
let plate_seed = params.seed + info.nearest_idx * 137u;
let basin_noise = snoise(raw_pos * 3.0 + seed_offset(plate_seed + 1000u));
let basin2 = snoise(raw_pos * 6.0 + seed_offset(plate_seed + 1010u)) * 0.5;
let basin3 = snoise(raw_pos * 12.0 + seed_offset(plate_seed + 1020u)) * 0.25;
let intra_plate_variation = (basin_noise + basin2 + basin3) / 1.75;

let variation_amplitude = select(1.5, 12.0, is_continental);
let thickness = base_thickness + intra_plate_variation * variation_amplitude;
```

### 6.3 Continental Margin Blending

```wgsl
let margin_width = 0.08;
let margin_blend = smoothstep(0.0, margin_width, boundary_dist);

let neighbor_thickness = select(7.0, 38.0, neighbor_continental);
let blended_thickness = mix(
    mix(thickness, neighbor_thickness + intra_plate_variation * select(1.5, 12.0, neighbor_continental), 0.5),
    thickness,
    margin_blend
);
```

### 6.4 Isostatic Conversion

```wgsl
let rho_crust = select(3.0, 2.7, is_continental);
let rho_mantle = 3.3;
let T_ref = 20.0; // reference thickness for sea level (km)

let h_km = (blended_thickness - T_ref) * (rho_mantle - rho_crust) / rho_mantle;
var height = h_km / 10.0; // scale to ~-0.4..+0.4 range
```

### 6.5 Boundary Features (Additive)

```wgsl
height += mountain_height * b_influence * convergence * ridge_mask;
height -= trench_depth * b_influence * convergence;
```

### 6.6 Hypsometric Shaping (Optional)

```wgsl
if (height > 0.0) {
    height = pow(height, 1.3) * 1.2; // most land near sea level, mountains rare
} else {
    height = -pow(abs(height), 0.85) * 1.1; // ocean floor mostly deep
}
```

### 6.7 Parameter Mapping from Current plates.wgsl

| Current                    | New Role                                   |
| -------------------------- | ------------------------------------------ |
| `plate_type` (0/1)         | Drives `base_thickness` (7 vs 38 km)       |
| `boundary_dist`            | Only for margin blending, NOT dome height  |
| `boundary_type`            | Same: drives convergent/divergent features |
| `mountain_scale`           | Same: scales boundary feature amplitude    |
| `boundary_width`           | Same: sigma for boundary influence spread  |
| `detail_scale`             | Same: fBm detail on top                    |
| NEW: `thickness_variation` | Controls amplitude of intra-plate noise    |

## 7. Impact Cratering Deep Dive

### 7.1 Pi-Scaling Framework (Holsapple-Schmidt)

Dimensionless pi-groups:

- `pi_D = D_crater / d_projectile * (rho_t / rho_p)^(1/3)` -- scaled crater diameter
- `pi_2 = (g * d_projectile) / v_i^2` -- gravity-scaled size
- `pi_3 = Y / (rho_t * v_i^2)` -- strength-scaled size
- `pi_4 = rho_t / rho_p` -- density ratio

**Gravity regime** (large craters, pi_2 >> pi_3):

```
pi_D = C_D * pi_2^(-mu_g)
```

**Strength regime** (small craters, pi_3 >> pi_2):

```
pi_D = C_D * pi_3^(-mu_s)
```

**Exponent mu values:**

- mu = 1/3 if crater size scales with momentum
- mu = 2/3 if crater size scales with energy (point-source limit)
- Typical values: mu ~ 0.41 (sand), mu ~ 0.55 (rock)

**Simplified energy scaling:**

- Gravity regime: D proportional to E^(1/4)
- Strength regime: D proportional to E^(1/3)

### 7.2 Simple vs Complex Crater Transition

The transition diameter scales inversely with surface gravity: `D_t ~ constant / g`

| Body                     | Gravity (m/s^2) | Transition Diameter   |
| ------------------------ | --------------- | --------------------- |
| Earth (sediment)         | 9.8             | ~2 km                 |
| Earth (crystalline rock) | 9.8             | ~4 km                 |
| Mars                     | 3.7             | ~3-8 km (avg ~5-8 km) |
| Moon (mare)              | 1.6             | ~14 km                |
| Moon (highland)          | 1.6             | ~17 km                |

### 7.3 Peak Ring and Multi-Ring Basin Thresholds

Progression with increasing size: central-peak crater -> peak-ring crater -> multi-ring basin.

| Morphology       | Onset Diameter (Moon)               | Onset Diameter (Mercury) |
| ---------------- | ----------------------------------- | ------------------------ |
| Peak-ring basin  | Largest onset in inner Solar System | 126 km (+33/-26)         |
| Multi-ring basin | >~300 km                            | --                       |

### 7.4 Late Heavy Bombardment

Occurred approximately **4.1 to 3.8 billion years ago** (some models extend to 4.2-3.5 Ga). Possibly triggered by giant planet migration (Nice model).

**Extrapolated cratering on Earth during the LHB:**

- ~22,000+ craters with diameter > 20 km
- ~40 impact basins with diameter ~1,000 km
- Several basins with diameter ~5,000 km

Impact velocities elevated from asteroid belt average of ~5 km/s to ~10 km/s. Melt volume increases **100-1,000x** when velocity doubles.

The bombardment rate decreased gradually from ~3.8 Ga until ~3.0 Ga (extended tail). Between 3.0 and 1.0 Ga: long periods (>600 Myr) of relative quiescence broken by shorter episodes (~200 Myr) of elevated flux.

### 7.5 Saturation Equilibrium

On ancient surfaces with low erosion, crater density reaches equilibrium: each new crater destroys approximately one existing crater of similar size.

- 5-10% of geometric saturation (theoretical maximum circle-packing density)
- Relative density (R) values between 0.1 and 0.3
- Saturation is size-dependent: small craters saturate at ~1% geometric saturation

### 7.6 Size-Frequency Distribution Details

The Neukum Production Function (NPF) is an 11th-degree polynomial in log(D), valid for D = 10 m to 300 km, revealing multi-slope structure:

- r < ~2 km: steep slope, cumulative exponent b ~ 3
- 2 km < r < ~30 km: shallow region (the "hump")
- Larger craters and basins: steep again
- D < 250 m: cumulative slope of -3.82
- D between 10-23 m: cumulative slope of -3.0

Two distinct impactor populations:

- **Population 1** (heavy bombardment): complex multi-sloped SFD similar to asteroid belt
- **Population 2** (since ~3.7 Ga): differential -3 single-slope power law

## 8. Erosion and Weathering Deep Dive

### 8.1 Stream Power Law Specifics

```
E = K * A^m * S^n
```

**Exponent constraints:**

- Concavity index: theta = m/n ~ 0.45-0.5 (Hack 1957)
- m ~ 0.3-0.5 (commonly ~0.5)
- n ~ 1-2 (commonly ~1, but observations suggest n > 1 for threshold-controlled incision)
- The ratio m/n ~ 0.5 is well-constrained from equilibrium river profiles

K varies by orders of magnitude: from ~10^-8 to 10^-3 depending on rock type and climate.

**Full form (Cordonnier et al. 2016):**

```
dh/dt = U(x) - K * A^m * |grad(h)|^n
```

Where U(x) is uplift rate. Typical: m=0.5, n=1.

### 8.2 Rock-Type Erosion Rates

| Factor                         | Erosion Rate        |
| ------------------------------ | ------------------- |
| Sedimentary outcrops           | 20 +/- 2.0 mm/kyr   |
| Metamorphic outcrops           | 11 +/- 1.4 mm/kyr   |
| Igneous outcrops               | 8.7 +/- 1.0 mm/kyr  |
| Temperate climates             | 25 +/- 2.5 mm/kyr   |
| Polar climates                 | 3.9 +/- 0.39 mm/kyr |
| Carbonate sedimentary rocks    | 5 mm/kyr            |
| Biotite-rich crystalline rocks | 1 mm/kyr            |
| Homogeneous crystalline rocks  | 0.2 mm/kyr          |

### 8.3 Chemical Weathering: Arrhenius Temperature Dependence

```
W = W_0 * exp(-E_a / (R * T))
```

**Activation energies for silicate dissolution:**

| Mineral                      | E_a (kJ/mol) | Notes                              |
| ---------------------------- | ------------ | ---------------------------------- |
| Generic silicates            | ~60          | 6x rate increase from 5 to 25 degC |
| Orthoclase                   | ~36          | Near-neutral pH                    |
| Plagioclase                  | ~107         | Near-neutral pH                    |
| Global average (humid sites) | 56 +/- 8     | Humidity index > 0.55              |

Practical effect: 3.5-9x increase in plagioclase dissolution rate as temperature rises from 3.4 to 22 degC. Chemical denudation rates: 0.2-5 mm per 1000 years in alpine environments.

### 8.4 Chemical Index of Alteration (CIA)

- Unweathered rock: CIA = 47
- Fully weathered: CIA = 100
- Chemical weathering dominates in warm, humid climates (tropical regions)
- Basaltic oceanic crust becomes less dense at ~15% per 100 Myr

### 8.5 Aeolian Erosion: Bagnold's Threshold

```
u*_t = A * sqrt((rho_s / rho_a - 1) * g * d)
```

Where A ~ 0.118 for Re\*\_p > 10.

**Key values:**

- Earth: upper limit on transported quartz grain diameter ~0.5 mm at ~30 m/s wind
- Earth: impact/fluid threshold ratio ~0.82
- Mars: impact threshold is ~one order of magnitude below fluid threshold (low atmospheric density)
- Mars: saltation, once initiated, is sustained at much lower wind speeds than required to start it

Nonfluvial subaerial erosion log-mean rate: **0.00032 mm/year** -- orders of magnitude slower than glacial (0.51 mm/yr) or fluvial (0.067 mm/yr).

### 8.6 Physical Weathering Parameters

**Frost wedging / ice segregation:**

- Water expands 9.2% upon freezing
- Theoretical max pressure: >200 MPa; realistic upper limit: ~14 MPa
- Granite tensile strength: ~4 MPa (easily exceeded)
- Ice segregation produces pressures up to 10x greater than simple frost wedging
- Most effective range: -4 to -15 degC

**Pressure release (exfoliation):**

- Differential stress in buttressed rock can reach 35 MPa

## 9. Sedimentation Models

### 9.1 Ocean Sedimentation Rates

| Environment                 | Sedimentation Rate             |
| --------------------------- | ------------------------------ |
| Abyssal clay (deep ocean)   | < 5 m/Myr (~0.005 mm/year)     |
| Pelagic biogenic sediments  | up to 200 m/Myr (~0.2 mm/year) |
| Continental shelf (typical) | 0.1-1 cm/year (1-10 mm/year)   |
| Near major river outlets    | 3-8 cm/year (30-80 mm/year)    |
| Active delta lobes          | 10-20 mm/year                  |

The Amazon River alone contributes over 1.2 billion tonnes of sediment annually. Coarse terrigenous sediment dominates near continental margins; fine pelagic clays and biogenic ooze dominate the deep ocean floor.

### 9.2 River Deltas

Deltas form when flow velocity decreases upon entering standing water. Coarser sediments (gravel, sand) settle near the delta front as bars and foreset beds; finer particles (silt, clay) travel farther offshore. Sediment delivery is strongly seasonal, with the largest loads transported during flood flow.

### 9.3 Sand Dune Migration

| Dune Type / Location          | Migration Rate (m/year) |
| ----------------------------- | ----------------------- |
| Kumtag Desert (average)       | 12.86                   |
| Kumtag (small-medium dunes)   | 13.84                   |
| Bodele Depression (median)    | 15.83                   |
| Namibia Sperrgebiet (average) | 7-32                    |
| Namibia (small, fast dunes)   | up to 83                |
| Namibia (large dunes)         | ~9                      |

Barchan dunes move proportionally to wind velocity and inversely proportionally to their height. Sediment moves along the stoss (windward) side to the crest; at critical overload, it avalanches down the lee slope.

### 9.4 Glacial Moraines

Moraines are found at glacier termini (terminal), alongside glaciers (lateral), between merging glaciers (medial), and beneath glaciers (ground). Glaciers deposit an extremely wide range of sediment sizes -- from fine clay to house-sized boulders (erratics) -- unsorted as **till**.

| Province                     | Volume                   | Area            | Age      |
| ---------------------------- | ------------------------ | --------------- | -------- |
| Deccan Traps (India)         | ~1,000,000 km^3          | ~500,000 km^2   | ~66 Ma   |
| Siberian Traps (Russia)      | 1,000,000-4,000,000 km^3 | ~5,000,000 km^2 | ~250 Ma  |
| Columbia River Basalts (USA) | ~175,000 km^3            | ~160,000 km^2   | ~17-6 Ma |

### 10.4 Magma Generation Mechanisms

1. **Decompression melting** -- Dominant at mid-ocean ridges; rising mantle rock melts as pressure drops.
2. **Flux melting (subduction)** -- Water from subducting crust lowers melting point of mantle wedge; generates arc volcanism.
3. **Hotspot / mantle plume** -- Deep plumes deliver anomalously hot material. Hotspot melt production during plume-ridge interaction is ~7x higher than after, due to thinner lithospheric coverage.

## 11. Open-Source Tectonic Implementations

### 11a. tectonics.js

- **Author:** Davidson (davidson16807)
- **URL:** https://davidson16807.github.io/tectonics.js/blog/
- **GitHub:** https://github.com/davidson16807/tectonics.js
- **Language:** JavaScript | **License:** CC-BY 4.0
- **Algorithm:** Tracks oceanic crust age to determine density. Identifies subduction regions, calculates velocity fields via vector calculus, uses image segmentation to identify plates. Island arcs emerge as natural behavior from 3D geometry.
- **Key technique:** Per-cell crust thickness (not just continental boolean), density varies with age. Elevation derived from thickness via isostasy.
- **Pipeline:** Age -> density -> subduction marking -> velocity field -> plate segmentation -> overlap removal + gap filling (mid-ocean ridges).

### 11b. PlaTec / plate-tectonics (Mindwerks fork)

- **Original:** Lauri Viitanen, Bachelor's thesis, Metropolia University, Helsinki
- **SourceForge:** https://sourceforge.net/projects/platec/
- **GitHub fork:** https://github.com/Mindwerks/plate-tectonics
- **Language:** C++ with Python bindings | **License:** LGPL-3.0
- **Description:** 2D terrain generator. Starts with flat fractal map, splits into plates, moves them. Continental collisions create mountain belts; oceanic-continental collisions cause subduction with coastal mountains/island chains.
- **Parameters (Python API):** seed, width, height, sea_level, erosion_period, folding_ratio, aggr_overlap_abs, aggr_overlap_rel, cycle_count, num_plates.
- **Used by:** WorldEngine, Widelands (open-source game).

### 11c. SimpleTectonics

- **Author:** Nick McDonald (weigert)
- **URL:** https://nickmcd.me/2020/12/03/clustered-convection-for-simulating-plate-tectonics/
- **GitHub:** https://github.com/weigert/SimpleTectonics
- **Language:** C++17 with GLSL shaders | **License:** MIT
- **Description:** GPU-accelerated clustered convection approach. Plates represented as Voronoi clusters of Poisson-sampled points. Points moved by convective heat-map forces. Collision detection via radial scanning; denser plate subducts.
- **Key parameters:** Domain 256x256, DT=0.025, cascade maxdiff=0.01, transport rate=0.2, dissolution=5% of growth.
- **Key technique:** Height computed per-pixel from properties of its Voronoi segment, not from a global dome function. Each segment carries its own height, blended at boundaries.

### 11d. World Orogen (planet_heightmap_generation)

- **Author:** raguilar011095
- **URL:** https://orogen.studio/
- **GitHub:** https://github.com/raguilar011095/planet_heightmap_generation
- **Language:** JavaScript (ES modules) | **License:** GPL-3.0
- **Description:** Comprehensive browser-based procedural planet generator. Features farthest-point seeding + flood-fill, convergent/divergent/transform boundary classification, stress-driven uplift, distance-field elevation, multiple erosion types, climate/biome simulation, and ocean currents.
- **Output:** Up to 65,536 px equirectangular maps.
- **Technical stack:** Three.js, Delaunator, 3D Simplex noise with fBm.

### 11e. Realistic Planet Generation and Simulation
